# ResearchGPT QA 检查清单

本文件为 ResearchGPT 的轻量级 QA 检查清单，用于在开发新功能或发布前快速确认现有能力未被破坏。

> **用法：** 每次合并重要功能或部署前，逐项勾选。自动化测试尚未覆盖的区域，以本清单 + [regression-tests.md](./regression-tests.md) 作为最低回归标准。

---

## 全站基础检查

- [ ] `npm run build` 通过，无 TypeScript 错误
- [ ] `npm run lint` 无新增阻塞性错误（如项目启用）
- [ ] 本地 `npm run dev` 可正常启动
- [ ] 未登录用户访问受保护页面时，行为符合预期（跳转登录或友好提示）
- [ ] 登录后可进入对话、文献追踪、文献库、文档翻译
- [ ] 页面标题、`<html lang="zh-CN">` 与全站中文描述正确
- [ ] 侧栏导航链接均可达，无 404
- [ ] 刷新页面后状态恢复合理（不丢失必要数据、不出现空白崩溃）
- [ ] 移动端 / 窄屏布局无严重错位（侧栏、表单、按钮可点击）

---

## AI 助手检查

- [ ] 可发送普通文本消息并收到回复
- [ ] 多轮对话上下文连贯
- [ ] 可上传图片并在对话中使用
- [ ] 可上传文档附件（PDF、Word 等，按产品支持范围）
- [ ] 对话历史侧栏可加载、切换、新建对话
- [ ] 导出回复功能可用（Markdown / 其他已支持格式）
- [ ] 空消息、过长消息、网络失败时有合理错误提示（中文）
- [ ] 流式输出中断时不导致页面卡死

---

## 文件上传检查

- [ ] 对话附件上传进度与成功/失败反馈清晰
- [ ] 不支持格式有明确中文错误提示
- [ ] 超大文件有合理限制或错误提示
- [ ] 上传失败后可重试，不污染后续请求
- [ ] 上传内容不会在 UI 中泄露服务端路径或内部实现细节

---

## 文档翻译检查

- [ ] 可上传 Word（.docx）并发起翻译
- [ ] 可上传 Excel（如产品支持）并发起翻译
- [ ] 翻译完成后可预览或下载结果
- [ ] 导出文件可正常打开，格式未严重损坏
- [ ] 翻译进行中按钮禁用 / 加载状态正确
- [ ] API Key 未配置或 provider 失败时有中文错误提示
- [ ] 翻译页文案遵循 [glossary.md](./glossary.md)

---

## 文献追踪检查

- [ ] 可填写研究方向、关键词、排除词、时间范围
- [ ] 点击「更新文献」后结果列表更新
- [ ] 关键词为空时无法提交更新
- [ ] 更新成功后显示成功提示与论文数量
- [ ] 更新失败时保留上次结果并显示中文错误信息
- [ ] 论文卡片显示标题、作者、摘要、相关度（分析后）、优先级标签
- [ ] 可忽略、标记已读、保存到文献夹
- [ ] 排序下拉（相关度、发表时间、被引用等）工作正常
- [ ] `DEBUG_LITERATURE=true` 时开发调试面板可见；默认关闭时不可见

---

## 文献库检查

- [ ] 文献库页面可打开并加载已保存论文
- [ ] 可按阅读状态筛选（新建 / 已收藏 / 已读 / 已忽略等）
- [ ] 可创建、重命名、删除文献夹（按产品支持范围）
- [ ] 可将论文移动到文献夹
- [ ] 取消收藏、恢复已忽略论文功能正常
- [ ] 文献库与文献追踪之间导航正常

---

## 论文详情页检查

- [ ] 从追踪器 / 文献库可进入论文详情
- [ ] 基本信息完整：标题、作者、摘要、发表时间、链接
- [ ] AI 工作区分析（若已生成）可正常展示
- [ ] 个人笔记可编辑并保存
- [ ] 阅读状态可更新
- [ ] 引用网络 / 参考文献 / 相关论文区块在数据可用时展示
- [ ] Semantic Scholar 限流（429）时显示友好提示，页面其余部分仍可用
- [ ] 导出 BibTeX / RIS / Markdown（按产品支持）可用
- [ ] **不修改论文详情 UI 布局时**，上述区块仍按预期工作

---

## 数据源检查

- [ ] 文献搜索按设计顺序调用 OpenAlex → arXiv → PubMed
- [ ] 各 provider 无结果时不导致整次搜索崩溃（除非全部为空）
- [ ] 返回论文带有正确来源标识（provider badges）
- [ ] 外部链接（原文、PDF）可打开且指向合理 URL
- [ ] 未实现的 provider（Crossref、DBLP 等）不会出现在用户可选来源中
- [ ] 不在 UI 中暴露原始数据库表名、内部 provider 实现细节

---

## 去重检查

- [ ] 同一 DOI / PMID / arXiv ID 的论文不会重复出现
- [ ] 标题高度相似的跨来源论文被合并为一条
- [ ] 合并后 provider badges 显示多个来源
- [ ] 合并后保留较长摘要、可用 PDF 链接、较早发表日期（按合并规则）
- [ ] 排除词过滤后列表无应被排除的论文

---

## UI 中文化检查

- [ ] 用户可见文案为简体中文（专有名词除外，见 [glossary.md](./glossary.md)）
- [ ] 保留英文：ResearchGPT、arXiv、PubMed、OpenAlex、Semantic Scholar、OpenReview、GitHub、DOI、PDF、BibTeX、RIS、Markdown 等
- [ ] 按钮、占位符、空状态、错误提示均为中文
- [ ] 无新增未翻译的英文 UI 字符串（除非为故意保留的专有名词）
- [ ] 日期、数字格式符合中文 locale（如被引用次数）

---

## 性能检查

- [ ] 文献更新请求在合理时间内完成或显示加载状态（不 silent hang）
- [ ] 大量论文列表滚动流畅
- [ ] 重复点击「更新文献」不会导致重复并发请求失控（按钮禁用有效）
- [ ] 论文详情页首屏加载可接受
- [ ] 无 obvious 内存泄漏（长时间切换页面后仍响应）

---

## 错误处理检查

- [ ] 网络错误、401、404、502 等有用户可读中文提示
- [ ] 错误提示不暴露 stack trace、SQL、API Key
- [ ] 文献更新失败后仍显示上次成功结果
- [ ] OpenAI / 外部 API 失败时不 corrupt 本地/数据库状态
- [ ] 表单校验错误聚焦到相关字段或给出明确说明

---

## 部署前检查

- [ ] `npm run qa`（即 `npm run build`）通过
- [ ] 环境变量文档齐全：OpenAI、Supabase、NCBI_EMAIL、OPENALEX_MAILTO 等
- [ ] 无 `.env`、密钥、token 提交到仓库
- [ ] 数据库 migration 已在目标环境执行（如有 schema 变更）
- [ ] [regression-tests.md](./regression-tests.md) 核心用例已手动执行
- [ ] [development-rules.md](./development-rules.md) 已阅读并遵守
- [ ] 变更范围与 PR 描述一致，未意外修改 chat / upload / translation / auth / 文献搜索核心逻辑
